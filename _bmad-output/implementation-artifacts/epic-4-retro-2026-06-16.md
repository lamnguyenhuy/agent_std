## What Went Well

- **Export pipeline end-to-end**: Stories 4.4 → 4.5 → 4.6 built cleanly on each other. Bundle gen → patch gen → feedback formed a natural progression with no rework.
- **Code review caught real issues**: Zip slip path traversal, 100 ms timeout race, stale closure patterns — all found and fixed before merge.
- **TDD on domain modules**: YAML serializer, diff generator, ZIP archive all had passing tests before workbench wiring. Zero regressions from wiring.
- **Architecture held**: `useMemo` derivation pattern continued from Epics 1-3 without deviation. No backend dependencies leaked in.
- **Side-effect boundaries**: `downloadBlob`, `ExportToast`, `useExportFeedback` kept pure UI — no business logic mixed in.

## What Could Be Improved

- **Duplicate translation calls**: `handleDownloadPatch` translated the playbook 2-3 times per click before code review caught it. More awareness of computation cost in event handlers.
- **Catch-and-silence pattern**: Stories 4.4 and 4.5 both deferred error handling to 4.6 with `catch { // Silently fail }`. This left a gap across 2 stories where users got zero feedback. Better to stub the feedback mechanism earlier or push it earlier in the sequence.
- **Determinism gap**: NFR-6 (deterministic generation) was noted in PRD but never enforced. `Date.now()` in archive names went through implementation and first code review before being flagged.
- **Component test infra**: Project had no React-testing-library setup. Had to add jsdom, jest-dom, and a vitest setup file in Story 4.6. Better to establish in Epic 1 or 2.

## Learning Patterns

- **LCS diff algorithm** chosen over external library. Simple, no dependency, but O(mn) memory. Acceptable for small generated files but won't scale to >500-line diffs.
- **JSZip 3.x API** uses `generateAsync` not `generateSync` — caught in tests. Browser ZIP generation without native APIs works but is async.
- **Vitest 4.x** removed `test.workspace` in favor of `test.projects`. Need to stay current with breaking changes in test infrastructure.
- **`@testing-library/react` with jsdom** requires explicit cleanup between tests when not using jest-circus. The `beforeEach(cleanup)` pattern is essential.
