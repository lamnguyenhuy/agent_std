## Challenges

1. **P1 UX bug deferred to next story instead of fixed inline** — Story 2-3 shipped with `if (nextText.trim().length === 0) return` (silent rejection). This is a UX failure (user edits disappear without feedback), not deferred enhancement. It was caught in code review but moved to Story 2-4.

2. **Accessibility missed on first pass in Story 2-4** — `aria-live` missing on error messages despite clear NFR-3 and UX-DR30 requirements (WCAG 2.2 AA, aria-live announcements). Caught in code review.

3. **Defensive coding not yet a habit** — `group.files?.map()` vs `group.files.map()` crash risk existed from Epic 1 through multiple stories before being patched in 2-4 code review.

4. **Edge cases not in spec, dev expected to know** — zero-width space handling in `trim()`, duplicate ID risk with `Date.now()`, and `aria-live` were all caught in code review, not in spec or dev notes. Three of four P1 patches in 2-4 were in this category.

5. **Technical debt accumulating** — 13 deferred items after Epic 2. Some are pre-existing and acceptable; others (aria-labels shift by index after deletion, `sourcePaths` schema violation) are accessibility and data integrity concerns.

---

## Key Insights

1. **Silent failures are not debt — they are regressions.** P1 UX bugs should be fixed in the same story, not deferred. The bar for "defer" is pre-existing issues, not issues introduced in the current story.

2. **Accessibility is a first-class requirement, not a polish pass.** NFR-3 and UX-DR30 are explicit. aria-live, aria-describedby, focus rings should be in task checklists, not discovered in code review.

3. **Defensive coding should be the default.** Optional chaining (`?.`) when accessing nested properties is a habit, not a case-by-case judgment call.

4. **String validation needs a richer mental model.** Any text input validation must consider: empty string, whitespace variants (`\n`, `\t`, `"   "`), zero-width characters (U+200B–U+200D, U+FEFF), and copy-paste edge cases. Not just `trim() === ""`.

5. **The `Result<T, AppError>` pattern works well.** AR-15 was correctly implemented. This pattern should be applied consistently to Epic 3 translator failures and Epic 4 export failures.
