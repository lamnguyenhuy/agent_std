## What Went Well

- **Foundation laid cleanly**: Next.js + shadcn/ui + Vitest + Playwright all established in Stories 1.1-1.2. No framework churn after.
- **Sample repo structure**: Fixed fixture data kept the demo deterministic and testable without filesystem access.
- **Scanner design**: `detectedFileGroups` pattern kept the scanner extensible for multiple file types (CLAUDE.md, .cursorrules, .claude/skills/, docs/).
- **Zod schema first**: AgentPlaybook schema defined upfront in Story 1.5, preventing data shape drift across all later epics.
- **Test infrastructure from day 1**: Vitest and Playwright set up in Story 1.2, paying off across all 20 stories.

## What Could Be Improved

- **Epic 1 retro deferred too long**: Writing this after all 4 epics done means lessons from Epic 1 are stale. Better to retro immediately after epic completion.
- **No project-context.md from the start**: `project-context.md` would have saved every agent session repeated context-loading time across 4 epics.
- **Fixture data coupling**: Sample repo fixtures baked into `src/lib/sample-repo/fixtures.ts` made tests deterministic but tightly coupled to specific file paths. Changing a fixture path breaks tests across multiple modules.

## Learning Patterns

- **`baseline_commit` in story frontmatter** proved valuable for generating clean diffs during code review. Should be standard practice.
- **Source-string tests** (`readFileSync` + `toContain`) were an effective lightweight alternative to React Testing Library for component contract validation. Good for MVP, but RTL needed later for interactive component tests (Story 4.6).
