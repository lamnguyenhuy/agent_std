# Decision Log: Agent Studio Product Brief

Created: 2026-06-09

## Decisions

1. **Product framing:** Agent Studio should be framed as a Git-native Agent Playbook system, not a generic prompt manager.
2. **Primary message:** Lead with "Review agent behavior changes like code."
3. **Emotional hook:** Acquisition should focus on loss of control -> restored control.
4. **MVP wedge:** Start with repo-level behavior, not a standalone skill library.
5. **Primary navigation:** Projects / Repos first; Playbook Library is secondary.
6. **Core primitive:** Agent Playbook.
7. **MVP Playbook shape:** Skills, Agents, Rules, Context, and Tool Translators.
8. **Deferred primitive:** Workflows are deferred to v2.
9. **Architecture:** MVP should be Git-native with `.agent-studio/playbook.yaml` in the repo.
10. **Tool Adapter language:** Use "Tool Translator" externally.
11. **Central aha moment:** Cross-Tool Preview.
12. **Review flow:** Plan + simplified Behavior Diff + downloadable patch/local preview.
13. **GitHub integration:** Actual GitHub PR creation is deferred to v1.1.
14. **2-week MVP repo strategy:** Use sample repo/demo workspace first, not arbitrary local repo support.
15. **Demo repo:** React/Next.js frontend repo.
16. **Scope cut:** Rules editable first; skills, agents, and context imported/read-only or lightly editable.

## Open Assumptions

1. Users will trust a Git-native Playbook as the source of truth more than native tool files.
2. Cross-tool output differences are meaningful enough to create a clear aha moment.
3. A controlled sample repo is sufficient for early validation before local repo support.
4. Downloadable patch/local preview is enough to prove reviewability before actual GitHub PR integration.
5. AI-forward tech leads are the strongest initial persona.
