# Decision Log: Agent Studio UX

Created: 2026-06-09

## Source Decisions

1. UX is created from PRD `_bmad-output/planning-artifacts/prds/prd-agent_std-2026-06-09/prd.md`.
2. MVP is a web demo workspace for a controlled React/Next.js Sample Repo.
3. Primary flow is Sample Repo -> Agent Playbook -> Rules edit -> Cross-Tool Preview -> Plan / Behavior Diff -> Patch Export.
4. Rules are the only fully editable Playbook section in MVP.
5. Skills, Agents, and Context are read-only or lightly editable.
6. No GitHub OAuth, no arbitrary local repo, no marketplace, no governance, no workflows.
7. Terms must match PRD Glossary: Agent Playbook, Tool Translator, Cross-Tool Preview, Plan, Behavior Diff, Patch Export.

## UX Authoring Decisions

1. Use Fast path from PRD; unresolved visual and component-system choices are tagged as `[ASSUMPTION]`.
2. Treat this as a focused developer tool, not a marketing landing page.
3. Primary surface is a desktop-first responsive web app; mobile is supported for inspection only, not the ideal demo surface.
4. Design assumes shadcn/ui + Tailwind conventions because the MVP is a modern web developer product. [ASSUMPTION]
5. Use Projects / Repos as the IA entry point, but collapse MVP into one Sample Repo workspace for the first demo.
6. Use a 3-panel desktop layout: Playbook structure, editor, generated outputs/review.
7. Use tabs for Tool Translator previews and Plan / Behavior Diff review.
8. Avoid decorative visuals, hero sections, or marketing composition inside the product UI.

## Open Assumptions

1. shadcn/ui + Tailwind is the intended UI system.
2. Desktop is the primary demo viewport.
3. A quiet dark-on-light dev-tool visual language is acceptable.
4. Users need only one key-screen mock for MVP planning; additional responsive mocks can be generated later.
